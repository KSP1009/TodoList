# Todo List 产品需求文档

## 1. 产品概述

本项目是一个本地化待办事项管理应用，面向需要高效管理每日任务的个人用户。产品参考示例网站的整体布局和交互：顶部为标题区，中间采用左侧统计与筛选、右侧任务添加与任务列表的双栏布局，视觉风格保持简洁、清晰、偏工具型。

应用支持创建、查看、筛选、搜索、编辑、删除和完成待办事项，并通过统计面板展示任务完成情况。所有数据存储在浏览器 LocalStorage 中，无需登录和后端服务。

## 2. 产品目标

- 帮助用户快速记录带有完整上下文的待办事项。
- 支持按分类、优先级、状态和关键词快速定位任务。
- 通过统计数据让用户了解当前任务压力和完成进度。
- 在本地浏览器中持久保存任务数据，刷新页面后数据不丢失。
- 使用现代前端技术栈实现清晰、可维护的单页应用。

## 3. 目标用户

- 需要管理学习、工作、生活任务的个人用户。
- 希望使用轻量工具记录待办事项，不希望依赖账号系统的用户。
- 需要通过优先级、截止日期、分类来安排任务的人。

## 4. 页面布局需求

### 4.1 整体布局

页面参考图片中的网站结构，采用居中内容区和双栏布局：

- 顶部标题区：
  - 主标题：待办清单名称，例如“我的待办清单”。
  - 副标题：简短说明，例如“Todo List · 高效管理每日任务”。
  - 标题区下方使用横向分隔线。
- 主体区域：
  - 左侧栏：统计数据卡片和筛选面板。
  - 右侧主区域：新增任务表单和任务列表。
- 页面背景：
  - 使用浅色背景，主体卡片使用白色或近白色。
  - 边框、阴影和间距参考图片中的简洁卡片风格。

### 4.2 左侧统计区域

统计卡片展示以下指标：

- 总任务数
- 待完成数量
- 已完成数量
- 完成率
- 可选增强：逾期任务数、今日截止任务数

统计数据需要根据当前 LocalStorage 中的任务实时计算。

### 4.3 左侧筛选区域

筛选面板支持：

- 按分类筛选：
  - 全部任务
  - 工作
  - 学习
  - 生活
  - 项目
  - 其他
- 按优先级筛选：
  - 全部优先级
  - 高
  - 中
  - 低
- 按状态筛选：
  - 全部状态
  - 待完成
  - 已完成

筛选项应具备清晰的选中态，参考图片中深色按钮表示当前筛选项。

### 4.4 新增任务区域

新增任务表单位于右侧顶部，参考图片中的横向输入区域，并扩展为支持完整字段。

新增任务字段：

- 标题，必填
- 描述，选填
- 截止日期，选填
- 优先级，必填，默认“中”
- 分类，必填，默认“工作”或“其他”

交互要求：

- 点击“添加”后创建任务。
- 标题为空时禁止添加，并给出表单提示。
- 添加成功后清空标题和描述，分类、优先级可保留默认值。
- 截止日期使用日期选择控件。

### 4.5 任务列表区域

任务列表位于右侧新增任务区域下方。

列表顶部包含：

- 区域标题，例如“今日任务”或“任务列表”。
- 搜索框，支持输入关键词快速搜索任务。

任务卡片内容：

- 完成状态复选框
- 任务标题
- 任务描述
- 分类标签
- 优先级标签
- 截止日期
- 编辑按钮
- 删除按钮

任务展示规则：

- 已完成任务标题可使用删除线或弱化颜色。
- 优先级需有明显视觉区分：
  - 高：橙色或红色强调
  - 中：蓝色强调
  - 低：灰色或绿色强调
- 截止日期使用 `date-fns` 格式化显示。
- 任务列表为空时展示空状态提示。

## 5. 功能需求

### 5.1 创建待办事项

用户可以通过新增任务表单创建待办事项。

验收标准：

- 输入标题、描述、截止日期、优先级、分类后可以成功创建任务。
- 创建后的任务立即显示在列表中。
- 刷新页面后任务仍然存在。

### 5.2 查看待办事项

用户可以查看所有待办事项，并看到每项任务的完整信息。

验收标准：

- 列表展示任务标题、描述、分类、优先级、截止日期和完成状态。
- 任务按创建时间或截止日期排序，推荐默认按创建时间倒序展示。

### 5.3 筛选待办事项

用户可以按分类、优先级、状态筛选任务。

验收标准：

- 分类筛选、优先级筛选和状态筛选可以组合使用。
- 切换筛选条件后，列表实时更新。
- 当前选中的筛选条件具有明确视觉状态。

### 5.4 搜索待办事项

用户可以通过搜索框快速查找特定任务。

搜索范围：

- 任务标题
- 任务描述
- 分类

验收标准：

- 输入关键词后，列表实时展示匹配结果。
- 搜索不区分大小写。
- 搜索可与分类、优先级、状态筛选组合使用。

### 5.5 编辑待办事项

用户可以编辑已有任务。

可编辑字段：

- 标题
- 描述
- 截止日期
- 优先级
- 分类
- 完成状态

验收标准：

- 点击编辑按钮后进入编辑状态或弹出编辑表单。
- 保存后列表和统计数据立即更新。
- 取消编辑时不修改原任务。

### 5.6 删除待办事项

用户可以删除不再需要的任务。

验收标准：

- 点击删除按钮后删除任务。
- 推荐弹出确认提示，避免误删。
- 删除后列表和统计数据立即更新。
- 刷新页面后被删除任务不再出现。

### 5.7 完成状态切换

用户可以通过复选框切换任务完成状态。

验收标准：

- 勾选后任务状态变为已完成。
- 取消勾选后任务状态恢复为待完成。
- 完成状态变化后统计数据实时更新。

### 5.8 统计功能

系统自动统计并展示任务数据。

统计指标：

- 总任务数
- 待完成数量
- 已完成数量
- 完成率
- 可选：逾期数量
- 可选：今日截止数量

完成率计算方式：

```text
完成率 = 已完成数量 / 总任务数 * 100%
```

当总任务数为 0 时，完成率显示为 0%。

## 6. 数据模型

### 6.1 Todo 类型

```ts
type Priority = 'high' | 'medium' | 'low';
type Category = 'work' | 'study' | 'life' | 'project' | 'other';
type TodoStatus = 'pending' | 'completed';

interface Todo {
  id: string;
  title: string;
  description?: string;
  dueDate?: string;
  priority: Priority;
  category: Category;
  status: TodoStatus;
  createdAt: string;
  updatedAt: string;
}
```

### 6.2 筛选状态

```ts
interface TodoFilters {
  category: Category | 'all';
  priority: Priority | 'all';
  status: TodoStatus | 'all';
  searchKeyword: string;
}
```

## 7. 状态管理需求

使用 Zustand 管理全局状态。

建议 Store 结构：

```ts
interface TodoStore {
  todos: Todo[];
  filters: TodoFilters;
  addTodo: (input: CreateTodoInput) => void;
  updateTodo: (id: string, input: UpdateTodoInput) => void;
  deleteTodo: (id: string) => void;
  toggleTodoStatus: (id: string) => void;
  setFilters: (filters: Partial<TodoFilters>) => void;
  clearFilters: () => void;
}
```

LocalStorage 持久化：

- 使用 Zustand persist 中间件，或自行封装 LocalStorage 读写。
- 推荐 LocalStorage key：`todo-list-storage`。
- 应处理 LocalStorage 中无数据或数据格式异常的情况。

## 8. 技术方案

### 8.1 技术栈

- 前端框架：React
- 开发语言：TypeScript
- 构建工具：Vite
- 状态管理：Zustand
- 样式方案：Tailwind CSS
- 日期处理：date-fns
- 数据存储：LocalStorage

### 8.2 推荐目录结构

```text
src/
  components/
    TodoForm.tsx
    TodoList.tsx
    TodoItem.tsx
    TodoFilters.tsx
    TodoStats.tsx
    EmptyState.tsx
  store/
    todoStore.ts
  types/
    todo.ts
  utils/
    date.ts
    todo.ts
  App.tsx
  main.tsx
```

### 8.3 日期处理

使用 `date-fns` 处理日期格式化和日期判断。

建议能力：

- 使用 `format` 格式化截止日期。
- 使用 `isBefore` 判断是否逾期。
- 使用 `isToday` 判断是否今日截止。

## 9. 视觉与交互要求

- 整体风格参考图片：简洁、清晰、卡片化、强调信息层级。
- 页面最大宽度建议为 `1200px` 至 `1440px`。
- 左侧栏宽度建议为 `280px` 至 `360px`。
- 右侧主内容区域自适应填充剩余空间。
- 表单、列表项、统计卡片保持统一边框、间距和字体层级。
- 移动端需要改为单列布局：
  - 顶部标题
  - 新增任务表单
  - 统计数据
  - 筛选面板
  - 任务列表

## 10. 非功能需求

- 页面刷新后数据不丢失。
- 基础交互响应应即时完成，无明显卡顿。
- TypeScript 类型定义完整，避免使用不必要的 `any`。
- 表单输入需要基础校验。
- 组件职责清晰，避免将所有逻辑堆叠在 `App.tsx`。
- 代码应便于后续扩展更多分类、排序和提醒功能。
